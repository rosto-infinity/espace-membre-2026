Voici les catégories classées dans un ordre d'apprentissage logique, partant des bases fondamentales jusqu'aux concepts avancés et à l'optimisation :

### 1. Php Essentials
**Pourquoi commencer ici :** C'est la base absolue. Avant de construire des architectures complexes ou des API, il est impératif de maîtriser la syntaxe, les variables, les boucles, les fonctions et l'interaction simple avec une base de données (PDO). C'est le prérequis pour tout le reste.

### 2. Php Oop Mastery
**Pourquoi ensuite :** Le PHP moderne est orienté objet. Une fois les bases procédurales acquises, il faut apprendre à structurer son code avec des classes, des objets, l'héritage et les design patterns. C'est indispensable pour comprendre les frameworks modernes et les sections suivantes.

### 3. Php Modern Features
**Pourquoi ensuite :** Après avoir compris la POO "classique", il est crucial d'apprendre les améliorations récentes du langage (PHP 8+) comme les Enums, les Attributes, les Union Types ou le constructeur promotionnel. Cela rend le code plus propre et plus performant.

### 4. Php Databases
**Pourquoi ensuite :** Bien que les bases de données soient introduites dans "Essentials", cette section approfondit l'interaction (Transactions, Optimisation, Query Builder). Il faut une bonne maîtrise de la POO pour comprendre des concepts comme l'Active Record ou les repositories abordés ici.

### 5. Php Testing
**Pourquoi à ce stade :** Avant de développer des applications critiques (API, Sécurité), il faut savoir garantir la qualité du code. Apprendre PHPUnit et le TDD (Test Driven Development) après la POO et les bases de données permet d'écrire du code fiable et refactorisable sereinement.

### 6. Php Security
**Pourquoi ici :** La sécurité devient critique une fois que l'on sait gérer des données et des utilisateurs. On aborde ici la protection contre les failles (XSS, SQL Injection, CSRF) et le chiffrement. C'est un prérequis obligatoire avant de développer des API exposées au monde extérieur.

### 7. Php Api Development
**Pourquoi ici :** Développer une API demande de cumuler les compétences précédentes : POO, Bases de données, Sécurité (Authentification) et Tests. C'est une application pratique concrète de toutes les compétences vues précédemment.

### 8. Php Ddd (Domain-Driven Design)
**Pourquoi après les API :** Le DDD est une approche architecturale avancée pour gérer la complexité métier. Il est difficile à appréhender sans avoir d'abord compris comment construire une application "classique" (API) et sans maîtriser parfaitement la POO et les Design Patterns.

### 9. Php Async
**Pourquoi ici :** La programmation asynchrone (ReactPHP, Fibers, Websockets) change radicalement le paradigme d'exécution de PHP (habituellement synchrone). C'est une technique avancée pour des besoins spécifiques de performance temps réel, qui nécessite une grande séniorité.

### 10. Php Performance
**Pourquoi en dernier :** L'optimisation (OPcache, JIT, Profiling) vient après le développement. On ne peut optimiser que ce qui est déjà construit et fonctionnel. Cela nécessite une compréhension profonde du fonctionnement interne du langage et de l'architecture.
